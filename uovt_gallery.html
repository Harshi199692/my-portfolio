UOVT Library — Management System

A full-featured web-based library management system built for the University of Vocational Technology. Designed with a warm academic theme, intuitive navigation, and an efficient workflow layout. Each screen was structured to help library staff manage books, members, circulation, and analytics with ease.

UI/UX HTML / CSS / JS Dashboard Design
01
Screen 01

Staff Login Page

The entry point of the system, designed to convey institutional authority while keeping the login process simple and secure.

  • Centred university logo with institution name and system title above the login card
  • Dark, textured background using a subtle grid pattern for depth
  • Clean white login card with a warm gold top-border accent
  • Username and password fields with inline icons and visible focus states
  • "Keep me signed in" checkbox for returning users
  • Full-width dark Sign In button with arrow indicator
02
Screen 02

Dashboard — Overview

The main hub of the system giving staff an instant snapshot of library activity, with quick-action tools built right into the view.

  • Left sidebar with grouped navigation: Overview, Catalog, Circulation, Members, and Analytics
  • Four summary stat cards at the top — Total Books, Available, Borrowed, and Members
  • Recent Activity feed showing time-stamped logs of borrows, returns, and registrations
  • Quick Issue panel on the right — member selector, book selector, and due date picker in one action
  • Overdue Summary widget showing count and daily fine rate at a glance
  • Logged-in admin name and Sign Out link anchored to the sidebar bottom
03
Screen 03

Books & Resources Catalog

A comprehensive and filterable book catalog allowing staff to view, search, and manage the entire library inventory in one place.

  • Filter tabs at the top — All, Available, Partially Borrowed, Unavailable
  • Book count indicator and a text filter input on the same row
  • Table layout with columns for Title & Author, ISBN, Category, Copies, Status, and Actions
  • Colour-coded status badges — Available (neutral), All Out (red-outlined)
  • Per-row action buttons: View and Issue — Issue button disabled when unavailable
  • + Add Book button in the top-right corner for quick catalog entry
04
Screen 04

Add New Book Form

A clean, structured data entry form for adding new titles to the library catalog with all necessary metadata fields.

  • Two-column layout for related fields — Author + ISBN, Publisher + Publication Year, Category + Number of Copies
  • Full-width Title field at the top as the primary required input
  • Category dropdown pre-populated with library classification options
  • Shelf Location field for physical placement reference
  • Description textarea for synopsis or librarian notes
  • Add to Catalog and Clear buttons at the bottom for quick form control
05
Screen 05

Issue Book — Circulation

The book issuing interface, designed for fast staff workflow with real-time visibility of all currently active borrows.

  • Left panel: Issue a Book form with Member, Book, Issue Date, Due Date, and Notes fields
  • Member and Book dropdowns show ID codes and availability count inline
  • Issue Date auto-populated; Due Date editable via date picker
  • Gold-coloured Issue Book submit button for clear call-to-action emphasis
  • Right panel: Currently Issued table with Member, Book, Due Date, and Status columns
  • Status badges colour-coded — Active (gold), Overdue (red) for quick identification
06
Screen 06

Returns & Fines

A dedicated screen for managing book returns and tracking outstanding fines, with overdue alerts prominently surfaced.

  • Alert banners at the top listing each overdue borrow with borrower name, due date, and calculated fine
  • Process a Return search panel — accepts Transaction ID, Member ID, or member name
  • Return History table on the right showing completed returns with date and fine paid
  • Fine amounts displayed as badge-style tags in warm orange for overdue entries
  • Clean two-column layout balancing the action form with the history log
07
Screen 07

Members Directory

A searchable and filterable directory of all registered library members shown as profile cards for quick identification.

  • Search bar at the top left for filtering by name, email, or member ID
  • Filter tabs — All, Students, Faculty, Staff — to segment the member list
  • Two-column card grid layout — each card shows avatar initials, full name, department, member type badge, member ID, and borrow count
  • Member type badges colour-coded — student (dark), staff (gold outline) for quick visual distinction
  • + Register button in the top-right for adding new members
08
Screen 08

Register New Member

A straightforward registration form for onboarding new library members with their personal and institutional details.

  • Two-column layout for paired fields — First Name + Last Name, Email + Phone, Member Type + Department
  • Member Type dropdown with options including Student, Faculty, and Staff
  • Full-width Address field spanning the form width
  • Required fields marked with asterisks for clear data validation guidance
  • Register Member (dark primary) and Clear (ghost) buttons at the bottom
09
Screen 09

Reports & Analytics

An analytics dashboard giving library administrators a data-driven overview of circulation, fines, and collection usage.

  • Tab navigation at the top — Overview, Circulation, Fines, Popular Books
  • Four summary stat cards — Total Transactions, Active Borrows, Fines Collected, Overdue count
  • Books by Category horizontal bar chart showing copies vs. availability for each subject area
  • Category labels and copy counts displayed inline for instant data readability
  • Gold-coloured bars with a muted background for clear visual contrast
